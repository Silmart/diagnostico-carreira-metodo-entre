# diagnostico-carreira-metodo-entre
Metodologia de diagnóstico de carreira, utilizando o método autoral ENTRE. 
Projeto: Metodologia de Diagnóstico de Carreira ("ENTRE Carreiras")
Link para a Ferramenta Ativa: https://entre-alma-carreira.lovable.app/

1. Objetivo do Projeto
Este projeto é a implementação de uma ferramenta de diagnóstico desenvolvida para mapear os desafios, perfis e necessidades de profissionais que procuram uma transição ou reavaliação de carreira.

O objetivo principal é transformar o conhecimento subjetivo de "aconselhamento de carreira" num fluxo lógico e estruturado, que possa guiar um utilizador a um insight acionável sobre a sua situação profissional.

2. A Metodologia por Trás do Questionário
O valor deste projeto não reside no código, mas na arquitetura da informação. A ferramenta foi construída com base numa metodologia de "árvore de decisão" que segmenta o percurso do utilizador.

A lógica foi desenhada para:

Qualificar o Utilizador: Entender o momento atual do profissional (Ex: desmotivado, à procura de promoção, transição, etc.).

Mapear Desafios: Utilizar perguntas condicionais para aprofundar as causas-raiz da insatisfação ou dos objetivos do utilizador.

Segmentar Perfis: As respostas do utilizador levam a diferentes "perfis" ou resultados de diagnóstico, oferecendo uma análise personalizada.

Validar Hipóteses: Cada pergunta foi formulada para testar uma hipótese específica sobre as necessidades do profissional.

3. Conexão com Treino de Modelos de IA
A criação deste diagnóstico demonstra diretamente as competências essenciais para o treino de modelos de IA, engenharia de prompts e curadoria de dados.

O processo de estruturar este questionário é análogo ao de treinar uma IA:

Estruturação de Conhecimento: Transformar um "domínio de especialista" (consultoria de carreira) em regras, fluxos e dados que uma máquina (ou um sistema lógico) pode entender e executar.

Engenharia de Prompt: O design de cada pergunta é semelhante a um "prompt", guiando o utilizador (ou a IA) para um resultado específico e útil.

Criação de Datasets: A lógica condicional e a segmentação de perfis são a base para criar "datasets" de treino (inputs e outputs) que ensinam um modelo a classificar e responder a situações complexas.

Antecipação de "Edge Cases": Pensar em todos os caminhos possíveis que um utilizador pode seguir no questionário é o mesmo que antecipar respostas e exceções num modelo de IA.

4. Tecnologias e Ferramentas
Plataforma: Lovable (Ferramenta No-Code para criação de questionários e lógica condicional)

Metodologias: Design de Experiência do Utilizador (UX), Mapeamento de Árvore de Decisão, Investigação de Utilizador.
